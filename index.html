Privacy Policy for Adobe Beacon Tracker Chrome Extension

Effective Date: May 6, 2025

Overview

The Adobe Beacon Tracker Chrome Extension (“the Extension”) is a client-side tool designed to help users audit Adobe Analytics beacon activity on websites they visit. This Privacy Policy outlines what data is (and is not) collected when using the Extension.

No Personal Data Collection

The Extension does not collect, store, transmit, or share any of the following types of information:

  • Personally identifiable information (PII)
  • Health or medical information
  • Financial or payment information
  • Authentication credentials
  • Personal communications
  • IP addresses or location data
  • Web browsing history
  • Clicks, keystrokes, or mouse movements
  • Text, images, or multimedia content from web pages

How the Extension Works

The Extension passively observes network requests made by the browser to Adobe Analytics endpoints (e.g., /b/ss/ or /v1/collect) and logs query string parameters such as pageName, eVar1, prop1, etc.

This data is:

  • Stored locally in your browser’s localStorage
  • Used only for display and CSV export
  • Never transmitted to any server or third party
  • Cleared entirely when the user clicks "Clear Logs" or runs clearAdobeBeacons()

Chrome Permissions

The Extension requests the following Chrome permissions:

host_permissions: ["<all_urls>"]

To detect and parse Adobe beacon requests on any site the user chooses to audit.

permissions: ["storage"]

To store the user's tracking toggle preference in chrome.storage.local.

These permissions are not used to collect any personal or website data.

Changes to This Policy

This policy may be updated from time to time. Updates will be posted on the extension's GitHub repository or accompanying website.
